# T0.1 — Repo + .NET solution scaffold

**Branch:** `chore/T0.1-solution-scaffold`  ·  **Depends on:** —  ·  **Status:** ⬜

## Context
Establish the .NET 10 solution with Clean Architecture projects and repo-wide quality config, so
every later task inherits the same structure, analyzers, and standards. This is foundation: it
must merge to `main` before any feature branch.

## Scope
**In:**
- Solution file + Clean Architecture projects: `Domain`, `Application`, `Infrastructure`, `Api`
  under `backend/src/`, and test projects `Domain.Tests`, `Application.Tests`,
  `Api.IntegrationTests` under `backend/tests/`.
- Project references per the dependency rule (HLD §4): `Api → Application → Domain`;
  `Infrastructure → Application, Domain`; `Domain` references nothing. Tests reference their target.
- `Directory.Build.props` (repo-wide): `net10.0`, `Nullable=enable`, `ImplicitUsings=enable`,
  `TreatWarningsAsErrors=true`, latest `LangVersion`, `EnableNETAnalyzers=true`, latest analysis level.
- `Directory.Packages.props` with Central Package Management (`ManagePackageVersionsCentrally=true`).
- `.editorconfig` encoding formatting + analyzer severities (the enforced source of truth).
- xUnit + FluentAssertions wired in each test project, with one passing sanity test per project.
- `backend/README.md` (or root README) skeleton: how to build/test the backend.

**Out:** business logic, EF Core/entities (T1.1), auth (T1.2), frontend (T0.2), Docker (T0.3),
the health endpoint (T0.3).

## Roadmap (high-level)
1. Confirm SDK is .NET 10 (`dotnet --version`).
2. Create the solution and the 4 src + 3 test projects with `TargetFramework=net10.0`.
3. Wire project references to satisfy the dependency rule; verify `Domain` has no project refs.
4. Add `Directory.Build.props`, `Directory.Packages.props`, `.editorconfig` at repo root (backend).
5. Add xUnit + FluentAssertions (versions in CPM); add one trivial passing test per test project.
6. `dotnet build` (zero warnings) and `dotnet format --verify-no-changes` must pass.
7. Write the backend README build/test section.

## Acceptance Criteria (Definition of Done — also workflow.md §6)
- [ ] `dotnet build` succeeds with **zero warnings** (warnings-as-errors active).
- [ ] `dotnet format --verify-no-changes` is clean.
- [ ] `dotnet test` runs and the sanity tests pass.
- [ ] Central Package Management is in effect (no inline `Version=` on PackageReferences).
- [ ] `Domain` project has no outgoing project references; dependency rule holds.
- [ ] Backend README explains build + test.
