## Context
Stand up the React/TypeScript frontend with strict linting/formatting, wire local pre-commit hooks
(changed-files-only), and add the GitHub Actions CI pipeline that gates every PR (backend +
frontend). After this lands, add the required CI status check to `main` branch protection.

## Scope
**In:**
- `frontend/` Vite + React + TypeScript app (TS `strict`), ESLint + Prettier configured and clean.
- Husky + `lint-staged` pre-commit hook: `eslint --fix` + `prettier --write` on staged frontend
  files; `dotnet format --include <staged>` on staged backend files (workflow.md ¬ß5).
- GitHub Actions CI (`.github/workflows/ci.yml`) running on PRs to `main`:
  - Backend job: restore ‚Üí build (warnings=errors) ‚Üí `dotnet format --verify-no-changes` ‚Üí `dotnet test`.
  - Frontend job: install ‚Üí typecheck ‚Üí `eslint .` ‚Üí `prettier --check .` ‚Üí build.
- Frontend README section (run dev server, build, lint).

**Out:** real UI/features (Epic 3), auth wiring (T3.1), Docker (T0.3). PR template already exists
(added at bootstrap) ‚Äî do not duplicate; extend only if needed.
