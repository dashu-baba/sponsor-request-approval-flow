# T3.2 — Requestor experience (UI)

**Branch:** `feat/T3.2-requestor-ui`  ·  **Depends on:** T3.1, T2.1, T2.4  ·  **Status:** ⬜

## Context
The requestor's screens: create/edit a request (with draft), view my requests on a dashboard,
see a request's detail + history, and upload supporting documents.
Visual spec: `docs/ui-mockups/dashboard.html`, `docs/ui-mockups/request-detail.html?role=requestor`.

## Scope
**In:**
- **Dashboard** (`dashboard.html` pattern): metric summary cards (total, drafts, pending, approved,
  rejected); sidebar status shortcuts with count badges; search + status/type filters; list/grid toggle;
  paginated request table — **only the requestor role shows draft metrics, draft filter, and draft rows**.
- Create/edit form in a **modal dialog** (React Hook Form + Zod mirroring backend rules: required fields,
  amount > 0, event date today-or-later); RequestorName read-only (from `/me`); optional Expected benefit.
- Save-as-draft vs submit actions; edit allowed only while Draft (mirror A1; rely on backend as source of truth).
- Row actions: Edit (draft only), Cancel (Draft + PendingManagerApproval per A2), View detail.
- Cancel confirmation modal; toast feedback on save/submit/cancel.
- Request detail: two-column layout (request fields + attachments left, workflow history timeline right).
- Supporting-document upload on detail (optional, multiple, drag-drop, type/size hints; backend enforces).

**Out:** approval queues (T3.3), admin (T3.4); draft visibility for other roles.

## Roadmap (high-level)
1. Build dashboard with metrics, filters, list/grid, and pagination (TanStack Query).
2. Build create/edit modal (RHF + Zod); map server validation errors.
3. Implement request detail + history timeline + attachment upload UI.
4. Wire submit/cancel with confirmation modal and toasts.
