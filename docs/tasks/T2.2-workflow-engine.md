# T2.2 — Workflow state machine + transitions + audit

**Branch:** `feat/T2.2-workflow-engine`  ·  **Depends on:** T2.1  ·  **Status:** ✅

## Context
The heart of the system: the approval state machine and its transition actions, with an immutable
audit trail and concurrency safety. Encodes the workflow + RBAC rules from the HLD.

## Scope
**In:**
- Domain state machine enforcing the allowed transitions + which role may perform each (HLD §6 table).
- Transition commands: `submit`, `cancel`, `approve`, `reject` as endpoints
  `POST /requests/{id}/{action}`.
- Rules: cancel only in Draft/PendingManagerApproval (A2); reject terminal (A3, A4); remarks
  **required on reject**, optional on approve (A5); no amount-based routing (A6); **no self-approval**
  (B4); SystemAdmin not in chain (B3).
- Append an immutable `WorkflowHistory` row on every successful transition (D1).
- Optimistic concurrency via `xmin`: concurrent transition → HTTP 409 (not double-transition).
- Invalid transition → 409; wrong role → 403.

**Out:** queues/history listing endpoints (T2.5), attachments (T2.4), frontend (T3.3).

## References
- HLD §6 (state machine + transition table), §15 (A2–A6, B3, B4, D1). Clarifications A2–A6, B3, B4, D1.
- `docs/best-practices/aspnetcore-best-practice-rules.md` (§24 idempotency/concurrency, §6 status codes).

## Roadmap (high-level)
1. Implement the transition table in the domain (from/action/role → to); reject illegal transitions with a domain exception.
2. Map domain exceptions to HTTP (invalid state → 409, wrong role → 403) via problem-details.
3. Implement submit/cancel/approve/reject command handlers enforcing A2/A3/A4/A5/B3/B4; write WorkflowHistory.
4. Apply optimistic concurrency (handle `DbUpdateConcurrencyException` → 409).
5. Add thin endpoints with role policies (Manager for manager step, FinanceAdmin for finance step).

## Acceptance Criteria (Definition of Done — also workflow.md §6)
- [ ] Build/format/test clean.
- [ ] Full Draft→Approved path works; reject (manager/finance) and cancel paths work and are terminal where specified.
- [ ] Remarks required on reject (A5); self-approval blocked (B4); SystemAdmin cannot approve (B3).
- [ ] Every transition writes a WorkflowHistory row; concurrent transition yields 409.

## Test Expectations
- Unit: every valid transition + a representative set of invalid ones (state + role); A2/A4/B4 rules.
- Integration: end-to-end Draft→Approved; manager-reject; finance-reject; cancel; **concurrency 409**;
  wrong-role 403.

## Best-Practices Checklist (subset)
- [ ] Domain invariants in the domain, not endpoints (csharp §30).
- [ ] Concurrency token enforced; correct status codes (aspnetcore §24, §6).
- [ ] Immutable audit trail (postgres/security audit rules).
