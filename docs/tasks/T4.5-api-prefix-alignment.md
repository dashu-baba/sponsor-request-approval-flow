# T4.5 ‚Äî API `/api` prefix alignment (dev + prod parity)

**Branch:** `fix/T4.5-api-prefix-alignment`  ¬∑  **Depends on:** T3.1  ¬∑  **Status:** ‚¨ú

## Context

The SPA page route `/requests/:id` and the REST API path `/requests/{id}` share the same URL
prefix. In **local dev**, Vite proxies every path starting with `/requests` to the backend. A
**browser refresh** on `http://localhost:5173/requests/7` therefore hits Kestrel (not the React
app) and returns **401 Unauthorized** ‚Äî the user sees a blank error page instead of the detail view.

Client-side navigation works because React Router handles the URL without a full document request.

In **Docker/production**, nginx already reverse-proxies only `/api/` to the API and serves the SPA
for all other paths ‚Äî so refresh on `/requests/7` works. However, the frontend HTTP client
currently calls unprefixed paths (`/requests`, `/auth`, `/me`, ‚Ä¶). Those `fetch` calls fall
through to the SPA fallback in nginx and never reach the API, so the composed stack cannot serve
authenticated data until this is fixed.

**Chosen approach:** prefix all browser API calls with `/api` and align the Vite dev proxy with
the existing nginx config (strip `/api` before forwarding). This fixes dev refresh, makes Docker
API calls work, and matches the architecture in HLD ¬ß10 and `docs/deploy.md`.

**Alternative considered (not chosen):** Vite proxy `bypass` for HTML `Accept` headers on
conflicting paths. Smaller diff (~10 lines in `vite.config.ts`) but leaves dev/prod routing
divergent and does not fix the production API proxy mismatch.

## Scope

**In:**

- **Frontend HTTP layer** ‚Äî centralize an `/api` base prefix in `auth-api.ts` (`requestJson`,
  `apiFetch`, and raw `fetch` for login/refresh/logout). Call sites in `requests-api.ts`,
  `admin-api.ts`, etc. keep unprefixed paths; the client prepends `/api` internally.
- **401 retry guard** ‚Äî keep the skip check on the **unprefixed logical path** passed to
  `apiFetch` (e.g. `path.startsWith('/auth/')`); prefixing happens only inside `fetchApi`.
- **Vite dev proxy** ‚Äî replace the six route-specific proxies (`/auth`, `/me`, `/health`,
  `/requests`, `/sponsorship-types`, `/system`) with a single `/api` entry that rewrites
  `/api/‚Ä¶` ‚Üí `‚Ä¶` on the backend (mirrors nginx `proxy_pass http://api:8080/`).
- **Refresh-token cookie path** ‚Äî change `AuthConstants.RefreshTokenCookiePath` from `/auth` to
  `/api/auth` so the browser sends the httpOnly cookie on `/api/auth/refresh` and
  `/api/auth/logout` (cookie `Path` must match the browser-visible URL, not the upstream path
  after nginx/Vite strip).
- **Tests** ‚Äî update `auth-api.test.ts` fetch URL expectations; update
  `AuthFlowTests` cookie path assertion.
- **Docs** ‚Äî short note in `docs/deploy.md` that local dev and Docker both route API traffic
  through `/api`.

**Out:**

- Moving backend endpoint registration under `/api` in Kestrel (proxy strip keeps backend paths
  unchanged).
- Renaming SPA routes (e.g. `/requests/:id` stays as the page URL).
- OpenAPI/Scalar URL changes (dev docs still on backend port `:5256` directly).
- TLS, Let's Encrypt, and production hardening (T4.3).
- Rate limiting or refresh-token reuse detection (backlog B-007, B-008).
