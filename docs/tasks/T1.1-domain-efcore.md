# T1.1 — Domain model + EF Core + initial migration

**Branch:** `feat/T1.1-domain-efcore`  ·  **Depends on:** T0.3  ·  **Status:** ⬜

## Context
Define the domain entities/enums and the EF Core persistence layer so every later feature has a
schema and DbContext to build on. This wires the `migrator` service (from T0.3) to a real migration.

## Scope
**In:**
- Domain entities: `SponsorshipRequest`, `SponsorshipType`, `WorkflowHistory`, `Attachment`; the
  `RequestStatus` enum (Draft, PendingManagerApproval, PendingFinanceReview, Approved, Rejected,
  Cancelled). Identity `User`/roles come in T1.2 — leave a `RequestorId` FK seam.
- `AppDbContext` (Infrastructure) + entity configurations: money `numeric(18,2)`, timestamps
  `timestamptz` (UTC), **snake_case** naming via `EFCore.NamingConventions`, **`xmin` concurrency
  token** on `SponsorshipRequest`, FKs + indexes for real query paths, audit columns.
- Initial EF migration; wire the `migrator` compose service to run `dotnet ef database update`.
- Connection config via options bound from env (no secrets committed).

**Out:** Identity/auth (T1.2), seeding (T1.3), request use-cases/endpoints (T2.x), API surface.

## References
- HLD §5 (domain model), §6 (statuses), §10 (migrator). Clarifications: C3 (numeric), D1 (history).
- `docs/best-practices/postgres-efcore-best-practice-rules.md` (types, constraints, indexes, migrations).
- Skill: `dotnet-data:optimizing-ef-core-queries`.

## Roadmap (high-level)
1. Add Domain entities + `RequestStatus` enum (no EF attributes; keep Domain dependency-free).
2. Add EF Core + Npgsql + `EFCore.NamingConventions` packages (CPM); create `AppDbContext`.
3. Add `IEntityTypeConfiguration` per entity: column types, snake_case, `xmin` concurrency token,
   FKs, indexes, audit columns.
4. Bind connection string from configuration/env; register `AddDbContext` with Npgsql.
5. Create the initial migration; confirm generated SQL is sane (review it).
6. Point the compose `migrator` service at `dotnet ef database update`; `docker compose up` applies it.

## Acceptance Criteria (Definition of Done — also workflow.md §6)
- [ ] Build/format/test clean; solution still green.
- [ ] Migration applies cleanly to a fresh Postgres (via `migrator`); tables use snake_case.
- [ ] `SponsorshipRequest` has an `xmin` concurrency token; money is `numeric(18,2)`; timestamps `timestamptz`.
- [ ] `Domain` project remains free of EF/infrastructure dependencies.

## Test Expectations
- Integration test (Testcontainers): the migration applies and the model round-trips a
  `SponsorshipRequest` (insert/read), confirming column types and concurrency token mapping.
