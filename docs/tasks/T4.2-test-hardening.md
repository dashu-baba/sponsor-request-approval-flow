# T4.2 — Testing pass to the HLD bar

**Branch:** `test/T4.2-test-hardening`  ·  **Depends on:** T3.4, T4.1  ·  **Status:** ⬜

## Context
Fill testing gaps so the suite meets the HLD's bar: strong unit coverage of the hard logic
(workflow + RBAC + validators) and integration coverage of the key API flows, all green in CI.

## Scope
**In:**
- Audit existing tests; add missing unit tests for the state machine (all transitions + invalid),
  RBAC/ownership rules, validators, and money/date handling.
- Add/round out integration tests (Testcontainers + WebApplicationFactory): full Draft→Approved,
  reject/cancel, concurrency 409, auth 401/403, attachments, queues visibility.
- Ensure determinism/isolation; confirm CI runs them and is green.
- Frontend: ensure key component tests exist for the critical flows.

**Out:** new product features; deployment (T4.3).

## Roadmap (high-level)
1. Run a gap analysis vs the HLD test bar; list missing cases.
2. Add unit tests for workflow/RBAC/validators (behavior, negative + edge cases).
3. Add/strengthen integration tests for the key flows incl. concurrency + authz.
4. Remove flaky/over-mocked tests; ensure isolation + determinism; confirm CI green.

## Acceptance Criteria (Definition of Done — also workflow.md §6)
- [ ] Build/format clean; full suite green in CI.
- [ ] State machine, RBAC, validators have direct unit tests (valid + invalid paths).
- [ ] Integration covers Draft→Approved, reject, cancel, concurrency 409, 401/403, attachments, queue scoping.
- [ ] No flaky/over-mocked tests; tests are deterministic and isolated.

- This task IS tests — the deliverable is the strengthened suite + green CI evidence.
