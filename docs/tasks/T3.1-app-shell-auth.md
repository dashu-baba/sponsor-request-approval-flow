# T3.1 — Frontend app shell + auth flow

**Branch:** `feat/T3.1-app-shell-auth`  ·  **Depends on:** T3.0, T1.2  ·  **Status:** ⬜

## Context
The React foundation: routing, providers, the typed API client, and the auth flow (login, token/
refresh handling, role context, route guards). Every later UI feature plugs into this shell.
Visual spec: `docs/ui-mockups/login.html` + shared layout in `docs/ui-mockups/app-shell.css`.

## Scope
**In:**
- App shell: router, layout (topbar + sidebar + main + footer), `QueryClient` provider, auth/role context.
- Login page per mockup; auth flow against `/auth/login` + `/auth/refresh` + `/me`; access token in memory,
  refresh via httpOnly cookie (no tokens in localStorage).
- Role-aware route guards and **role-specific sidebar nav** (see mockups):
  - **Requestor:** Dashboard, My Requests, Drafts, Pending, Approved, Rejected
  - **Manager / Finance:** Dashboard only (+ Account stub)
  - **Admin:** Dashboard, Sponsorship Types (+ Account stub)
- Post-login redirect to the role dashboard (`dashboard.html` equivalents).
- User chip (avatar, name, role from `/me`); sidebar Sign out.
- Typed API client + base query hooks; Zod schemas for auth responses.
- Loading/empty/error/success baseline per `loading-state.html` / `error-state.html`; error boundary around major sections.

**Out:** request/approval/admin feature views (T3.2–T3.4); backend (Epic 1/2); Profile, notifications,
Settings, Remember me, Forgot password (shown as stubs/disabled in login mockup only).

## Roadmap (high-level)
1. Set up router + providers + layout matching mockup shell; auth/role context.
2. Build the typed API client (handles 401 → refresh → retry; logout on refresh failure).
3. Implement login page + `/me` bootstrap; store access token in memory.
4. Add role-aware route guards + nav; Zod-validate auth responses.
5. Establish loading/empty/error/toast + error-boundary patterns.

## Acceptance Criteria (Definition of Done — also workflow.md §6)
- [ ] Typecheck/ESLint/Prettier/build clean.
- [ ] Login works against the API; refresh-on-401 works; logout clears session.
- [ ] Guards redirect unauthenticated users; role context available to features.
- [ ] Shell layout and role nav match `docs/ui-mockups/`; no tokens in localStorage.
