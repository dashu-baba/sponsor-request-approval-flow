# T3.4 — System Admin console (UI)

**Branch:** `feat/T3.4-admin-ui`  ·  **Depends on:** T3.1, T2.3, T2.5  ·  **Status:** ✅

## Context
The System Admin's screens: view all submitted requests on the **admin dashboard**, inspect any
request's workflow history (read-only), and manage the sponsorship-type lookup.
Visual spec: `docs/ui-mockups/dashboard-admin.html`, `docs/ui-mockups/admin-sponsorship-types.html`,
`docs/ui-mockups/request-detail.html?role=admin`.

## Scope
**In:**
- **Admin dashboard** (`dashboard-admin.html`): submitted-requests-only list with metrics (no drafts),
  info banner, search, status/type filters, list/grid toggle, pagination, View → read-only detail.
  **No separate all-requests page** — this dashboard is the sole org-wide request view.
- Request detail + full workflow history (read-only; admin is not in the approval chain — B3).
- **Sponsorship-type CRUD** (`admin-sponsorship-types.html`): table with active-request counts,
  add/edit modal, delete confirmation, delete-blocked-when-in-use feedback.
- Admin nav: Dashboard + Sponsorship Types only.

**Out:** approval actions (admin can't approve — B3); draft visibility (B5); requestor/reviewer screens (T3.2/T3.3).
