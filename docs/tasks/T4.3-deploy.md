# T4.3 — Deployment finalize + live bring-up

**Branch:** `chore/T4.3-deploy`  ·  **Depends on:** T4.2  ·  **Status:** ⬜

## Context
Harden the Docker Compose stack for a real single-VPS deployment (TLS, limits, env, migrator) and
write the runbook so the live URLs (frontend, API, docs) can be brought up.

## Scope
**In:**
- Finalize `docker-compose.yml`: production-leaning settings, the one-shot `migrator`, healthchecks,
  named volumes, internal network, only nginx published.
- nginx: serve SPA + reverse-proxy `/api` + docs, TLS via Let's Encrypt, `client_max_body_size`
  matching the upload cap, security headers, HTTP→HTTPS.
- `.env.example` complete; production secrets supplied at deploy time (not committed).
- `docs/deploy.md` runbook: provision VPS, point DNS, issue cert, `docker compose up`, smoke check, rollback.

**Out:** feature work; the actual VPS/domain provisioning (the human does that following the runbook).

## References
- HLD §10 (deployment). `docs/best-practices/deployment-docker-best-practice-rules.md` (whole file).

## Roadmap (high-level)
1. Finalize Dockerfiles (multi-stage, non-root) and compose (healthchecks, depends_on conditions, volumes, network).
2. Configure nginx: SPA + proxies, TLS/Let's Encrypt, body-size limit, security headers, redirects.
3. Complete `.env.example`; document required secrets.
4. Write the end-to-end `docs/deploy.md` runbook incl. smoke test + rollback.
5. Validate locally with `docker compose up` (full stack healthy, migrations applied, app reachable).

## Acceptance Criteria (Definition of Done — also workflow.md §6)
- [ ] `docker compose up` brings the full stack up healthy; migrator applies migrations; app reachable through nginx.
- [ ] TLS/headers/body-limit configured; only nginx publishes a port; non-root API.
- [ ] `docs/deploy.md` lets someone provision from zero; `.env.example` complete, no secrets committed.

## Test Expectations
- Manual: full `docker compose up` smoke (login, create→approve a request, open docs URL) — capture in PR evidence.

## Best-Practices Checklist (subset)
- [ ] Multi-stage/non-root/pinned images; healthchecks; migrator one-shot; named volumes (deployment §1–12).
- [ ] TLS at proxy, security headers, body limits; secrets not baked in (deployment §6–7,14–16).
