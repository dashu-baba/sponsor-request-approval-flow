# T3.3 — Manager + Finance approval queues (UI)

**Branch:** `feat/T3.3-approvals-ui`  ·  **Depends on:** T3.1, T2.2, T2.5  ·  **Status:** ✅

## Context
The reviewer experience: managers and finance admins work from their **role dashboard** (no separate
queue route), open a request, and approve/reject with remarks.
Visual spec: `docs/ui-mockups/dashboard-manager.html`, `docs/ui-mockups/dashboard-finance.html`,
`docs/ui-mockups/request-detail.html`, `docs/ui-mockups/concurrency-conflict.html`.

## Scope
**In:**
- **Role dashboard** per mockup (metrics without drafts, search, status/type filters, list/grid,
  pagination) scoped to submitted requests only — **no draft rows, draft metrics, or Draft filter** (B5).
- Manager queue: `PendingManagerApproval`; Finance queue: `PendingFinanceReview`.
- Requestor column on reviewer tables; inline Approve/Reject on actionable rows **and** on detail page.
- Approve/reject modals: **remarks required on reject** (A5), optional on approve.
- Request detail with workflow history timeline; attachments read-only for reviewers.
- Optimistic-concurrency feedback per `concurrency-conflict.html`: 409 → warning banner, disabled actions,
  refresh/view-latest CTA.
- 403 surfaced via error banner pattern; toast + list refresh on success.

**Out:** requestor screens (T3.2), admin (T3.4); draft visibility; the transition backend (T2.2).

## Roadmap (high-level)
1. Build manager and finance dashboards (role-scoped lists from T2.5; no drafts).
2. Wire list + detail approve/reject modals; enforce remarks-on-reject (A5).
3. Handle 409 concurrency + 403 gracefully; invalidate queries on success.
4. Show history timeline for context on detail view.
